3.2 Déplacer et copier

Quand tu écris let s1 = String::from("hello"); puis let s2 = s1;, la valeur est déplacée. s1 n'est plus utilisable. Cela s'appelle un move. En revanche, les types simples comme i32, bool ou f64 implémentent le trait Copy : ils sont dupliqués automatiquement et les deux variables restent valides. Pour garder s1 utilisable avec un String, il faudra cloner explicitement avec .clone(), ce qui copie vraiment les données en mémoire. Choisir entre move et clone évite les copies surprises.
